[I18N-01] CSV parser: split on "," — fields containing commas are silently broken
  File: I18nAudit.java:32-33
  Code: String[] row = s.split(",", 4); if (row.length >= 4) out.put(row[0], new String[]{row[1],row[2],row[3]});
  Problem: If a CSV field value contains a comma (e.g., "New, Improved" as a translation),
           split(",", 4) would produce more than 4 fields and silently discard the value.
           More importantly, row.length is checked as >= 4, but if the split produces
           exactly 4 parts and one of the translations contains a comma, the last part
           gets truncated. For I18n keys, translations should be quoted or the parser
           should use a proper CSV library.
  Fix: Use a proper CSV parser (e.g., opencsv) or handle quoted fields:
         String[] row = s.split(",(?=(?:[^\"]*\"[^\"]*\")*[^\"]*$)", -1);
